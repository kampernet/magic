      this is a class for applying Aspect Oriented Programming in PHP it does this by maintaining an internal array of the lines of code and eval()'ing the modified ( ie: aspects injected ) code and instantiating an internal object.

      it then provides access to that objects methods, with advice applied at the appropriate pointcuts, via __call.

      please note this class relies on the original class being well formatted code to run. it must be multi-line code and have the class endbraces on different lines than the method endbraces. the method endbraces must be on their own line.

      what I mean to say is it doesn't use the tokenizer
